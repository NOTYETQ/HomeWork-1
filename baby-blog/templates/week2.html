Week 2 - Web Applications & HTTP

In week 2, we explored how web applications work, focusing on how requests interact with handlers and the role of HTTP.

Client-Server Communication

Web applications rely on request-response cycles where clients send requests to a server, which then processes them and sends responses.

//code showing how handlers work
            package main

            import (
                "log"
                "net/http"
            )
            
            // a handles function name 'home'
            func home(w http.ResponseWriter, r *http.Request) {
                w.Write([]byte("Hello from Flash"))
            }
            
            func info(w http.ResponseWriter, r *http.Request) {
                w.Write([]byte("Bryan Hernandez"))
            }
            
            func color(w http.ResponseWriter, r *http.Request) {
                w.Write([]byte("Blue"))
            }
            
            func user(w http.ResponseWriter, r *http.Request) {
                w.Write([]byte("User Bryan Hernandez"))
            }
            
            
            func main() {
            
                // mux is our router
                mux := http.NewServeMux()
                // the route pattern/endpoint/URL path
                mux.HandleFunc("/", home)
               
                mux.Handle("/user", user)
            
                mux.HandleFunc("/info", info)
            
                mux.HandleFunc("/color", color)
            
                log.Print("starting server on :4000")
            
                // start a local web server
                err := http.ListenAndServe(":4000", mux)
                log.Fatal(err)
            
            }

In the code above we can see how there are multiple handler functions. In a Web application there are multiple handlers to handle all the different types of requests.
